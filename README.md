## Overview

Pakistan receives over **$26 billion** in annual worker remittances — the single largest source of foreign exchange. This project builds a full forecasting system to predict monthly remittance inflows using macroeconomic signals including oil prices, the USD/PKR exchange rate, Gulf employment levels, and political stability.

**Three models** are trained and compared:

| Model | Type | Features Used |
|-------|------|---------------|
| ARIMA/SARIMA | Statistical baseline | Past remittances only |
| XGBoost | Gradient boosting | All 26 engineered features |
| LSTM | Deep learning (BiLSTM) | 12-month sliding windows |

## Quick Start

### 1. Clone the repository

```bash
git clone https://github.com/YOUR_USERNAME/pakistan-remittance-predictor.git
cd pakistan-remittance-predictor
```

### 2. Create a virtual environment (recommended)

```bash
python -m venv venv

# Activate:
# Windows:
venv\Scripts\activate
# macOS / Linux:
source venv/bin/activate
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

> **Note — TensorFlow:** If you just want to explore without deep learning, comment out `tensorflow` in `requirements.txt`. Step 6 will automatically fall back to Holt-Winters Exponential Smoothing.

> **Apple Silicon (M1/M2/M3):** Replace `tensorflow` with `tensorflow-macos` and `tensorflow-metal`.

### 4. Run the full pipeline

```bash
python step1_collect_data.py    # Build dataset
python step2_eda.py             # EDA + 5 charts
python step3_features.py        # Feature engineering
python step4_arima.py           # ARIMA model
python step5_xgboost.py         # XGBoost model
python step6_lstm.py            # LSTM model
python step7_compare.py         # Final comparison
```

### 5. Open the dashboard

Just open `dashboard.html` in any modern browser — no server required.

```bash
# macOS
open dashboard.html

# Windows
start dashboard.html

# Linux
xdg-open dashboard.html
```

## Pipeline Steps

### Step 1 — Data Collection (`step1_collect_data.py`)
Builds a 180-month dataset (Jan 2010 – Dec 2024) using real published anchor values from:
- **State Bank of Pakistan** workers' remittance reports
- **U.S. EIA** Brent crude oil monthly averages
- **SBP / IMF** USD/PKR exchange rate history
- **World Bank** Gulf employment and political stability proxies

Also includes `step1_live_api.py` which fetches live/recent data from the EIA and World Bank APIs to extend the dataset beyond 2024.

### Step 2 — Exploratory Data Analysis (`step2_eda.py`)
Generates 5 publication-quality charts:
1. Remittance trend with annotated events (COVID, oil crash, political crises)
2. All feature signals dashboard
3. Correlation heatmap
4. **Cross-correlation oil price lag analysis** — key finding: oil prices lead remittances by ~3 months
5. Seasonal pattern by month (Eid effect visible)

### Step 3 — Feature Engineering (`step3_features.py`)
Creates 26 ML features across 4 categories:
- **Lag features** — oil price at 1m, 2m, 3m, 6m delay; remittance lags
- **Rolling statistics** — 3-month and 6-month averages, PKR volatility
- **Momentum signals** — percentage changes, direction dummies
- **Calendar features** — month sin/cos encoding, Eid dummy, post-COVID flag

Performs **chronological 80/20 train/test split** to prevent data leakage.

### Step 4 — ARIMA Baseline (`step4_arima.py`)
- ADF stationarity test → auto-differencing
- Grid search over ARIMA/SARIMA orders
- 95% confidence intervals
- Saves forecasts + metrics to `models/arima_results.json`

### Step 5 — XGBoost (`step5_xgboost.py`)
- TimeSeriesSplit cross-validation (5 folds, no shuffle)
- Hyperparameter tuning
- Feature importance ranking (SHAP-style)
- Saves forecasts + feature importance to `models/xgboost_results.json`

### Step 6 — LSTM (`step6_lstm.py`)
- Bidirectional LSTM with 12-month input windows
- Early stopping + learning rate reduction
- Automatic Holt-Winters fallback if TensorFlow is not installed
- Saves forecasts + metrics to `models/lstm_results.json`

### Step 7 — Comparison (`step7_compare.py`)
Loads all three model results and produces:
- Side-by-side metrics table (MAE, RMSE, MAPE, R²)
- All three forecasts overlaid on actual values
- MAE comparison bar chart
- Printed summary report

## Model Results

*(Results from demo data — run the pipeline to see your actual results)*

| Model | MAE | MAPE | R² |
|-------|-----|------|-----|
| ARIMA/SARIMA | ~$195M | ~7.8% | ~0.88 |
| XGBoost | ~$112M | ~4.4% | ~0.96 |
| LSTM (BiLSTM) | ~$148M | ~5.9% | ~0.93 |

**Key Finding:** XGBoost outperforms the baseline by ~40% on MAE. The top predictors are `remit_lag_1m`, `usd_pkr`, and `oil_lag_3m` — confirming that oil prices lead remittance changes by ~3 months.

## Configuration

### EIA API Key

The dashboard has the EIA key pre-configured. To use your own key in the Python scripts (`step1_live_api.py`), edit line 49:

```python
EIA_API_KEY = "YOUR_KEY_HERE"  # get free key at eia.gov/opendata
```

### Extending the Dataset

To pull live data beyond December 2024, run:

```bash
python step1_live_api.py
```

This fetches the latest available data from EIA and World Bank APIs and appends it to your dataset.

### Changing the Forecast Horizon

In `step3_features.py`, adjust the train/test split:

```python
split_idx = int(len(df_clean) * 0.80)  # change 0.80 to desired split
```
